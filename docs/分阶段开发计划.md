# AI English Partner 前端迭代分阶段计划

> 基于 `docs/react-native-app-development.md`、`docs/接口文档.md` 和当前 React Native 已实现页面制定。  
> 当前范围：只迭代前端。后端、真实 REST、真实 WebSocket、真实音频采集与播放在后续阶段对接，本计划只做页面 UI、状态、mock 数据和接口预留。

## 1. 当前开发边界

### 当前负责

- 基于当前已完成的 React Native 页面继续迭代。
- 建立前端设计系统。
- 建立前端类型模型。
- 建立 mock 数据。
- 建立页面导航和状态流。
- 完善 `ScenarioSelectScreen`、`PracticeScreen`、`SessionSummaryScreen`、`HistoryScreen` 四个现有页面。
- 完成 loading、empty、error、success 等 UI 状态。
- 预留 REST / WebSocket / 音频 client interface，但不接真实后端。

### 当前不负责

- 不实现后端服务。
- 不直连豆包、腾讯智聆、ASR、TTS 等供应商。
- 不实现真实 WebSocket 对话。
- 不实现真实麦克风采集。
- 不实现真实 AI 音频播放。
- 不实现真实登录、token 刷新、配额、报告生成。

### 后续对接后端时再做

- 将 mock client 替换为真实 REST client。
- 将 mock realtime client 替换为真实 WebSocket client。
- 接入真实麦克风权限、音频采集、音频发送和 AI 音频播放。
- 接入真实报告、历史、画像和推荐接口。

## 2. 前端开发链路

```text
当前页面结构确认
  -> 工程基线整理
  -> 设计系统
  -> 前端类型模型
  -> Mock 数据
  -> 导航状态机
  -> 场景选择页面完善
  -> 实时对话 Mock 页面
  -> 课后总结 Mock 页面
  -> 历史记录 Mock 页面
  -> 前端 client interface
  -> 页面状态完善
  -> 可访问性与移动端适配
  -> 前端 mock 全链路验收
  -> 后端对接交接文档
```

## 3. 阶段计划

### 阶段 0：当前页面结构确认

目标：确认后续开发以当前 React Native 已实现页面为基准。

交付内容：

- 确认当前页面为：`ScenarioSelectScreen`、`PracticeScreen`、`SessionSummaryScreen`、`HistoryScreen`。
- 确认当前导航入口为 Expo Router `src/app/index.tsx` -> `AppNavigator`。
- 确认主流程为：场景选择 -> 实时对话 -> 课后总结，历史记录通过底部 tab 进入。

验收门禁：

- 不再引用 HTML 原型作为页面实现依据。
- 继续保留当前四页 tab 结构。
- 页面职责、入口、返回路径全部确认。

### 阶段 1：工程基线

目标：整理当前 Expo 前端工程边界。

交付内容：

- 保留 Expo Router 入口。
- 保留当前页面导航引用。
- 基于现有结构补齐目录：`screens`、`components`、`types`、`data`、`clients`、`state`、`constants`。
- 建立命名规则和导入路径规则。

验收门禁：

- App 可以启动到当前四页 tab 页面。
- 页面入口不依赖 HTML 原型。
- 不引入真实后端依赖。
- `pnpm lint` 和 `pnpm exec tsc --noEmit` 通过。

### 阶段 2：设计系统

目标：建立前端视觉规范和基础 UI token。

交付内容：

- 颜色、字号、间距、圆角、阴影 token。
- 页面容器、按钮、标签、卡片、列表项、状态条基础样式。
- 将当前页面色彩集中映射到 React Native token。

验收门禁：

- 主色、深色训练舱、提示色、危险色集中维护。
- 页面颜色不散落在业务组件中。
- 按钮文本对比度可读。
- 不接真实接口。

### 阶段 3：前端类型模型

目标：先定义页面和 mock 数据所需的前端类型。

交付内容：

- `Scenario`。
- `Persona`。
- `TrainingGoal`。
- `PracticeSessionDraft`。
- `TranscriptTurn`。
- `SessionReport`。
- `CorrectionItem`。
- `PronunciationIssue`。
- `PracticeHistoryItem`。
- `ExpressionAsset`。
- `RecommendationTopic`。

验收门禁：

- 类型字段参考接口文档，但不强制依赖真实接口返回。
- mock 数据和页面 props 使用同一批类型。
- 不使用 `any` 作为主要业务数据类型。

### 阶段 4：Mock 数据

目标：用 mock 数据支撑首版前端全流程。

交付内容：

- 今日推荐 mock。
- 场景 mock。
- AI 角色 mock。
- 实时字幕 mock。
- 音量状态 mock。
- 报告 mock。
- 历史和表达资产 mock。

验收门禁：

- Mock 字段与阶段 3 类型一致。
- Mock 可以支撑从首页到复盘再到资产库的完整流程。
- 没有乱码文案。
- 不依赖网络。

### 阶段 5：导航状态机

目标：建立纯前端页面流转。

交付内容：

- 页面状态：`practice`、`conversation`、`summary`、`history`。
- 当前场景、当前会话草稿、当前报告的前端状态。
- 页面跳转方法：选择场景、开始训练、结束进入总结、复练、查看历史。

验收门禁：

- 可以在 4 个页面之间按主流程切换。
- `TabKey` 与当前四页结构一致。
- 页面切换不会丢失当前选择的场景。
- 不调用真实后端。

### 阶段 6：场景选择页面

目标：完善当前场景选择与练习配置页。

交付内容：

- 场景列表。
- 今日建议卡片。
- 难度选择。
- 推荐原因。
- 进入实时对话按钮。

验收门禁：

- 首屏清楚回答“练什么”和“为什么练”。
- 主操作只有一个。
- 无推荐数据时有默认建议。
- 页面完全由 mock 数据驱动。

### 阶段 7：练习配置完善

目标：在当前场景选择页上补齐练习前配置。

交付内容：

- 场景详情。
- AI 角色选择。
- 难度、口音、语速、反馈强度选择。
- 必练表达列表。
- 进入实时对话按钮。

验收门禁：

- 至少可以选择一个场景并生成完整前端会话草稿。
- 配置变化不导致布局跳动。
- 开始按钮只在配置完整后可用。
- 不调用真实 `POST /practice-sessions`。

### 阶段 8：实时对话 Mock 页面

目标：实现不接真实音频、不接真实 WebSocket 的实时房间。

交付内容：

- 深色训练舱页面。
- mock 连接状态。
- mock 会话计时。
- 用户字幕。
- AI 字幕。
- mock 音量条。
- mock 教练提示。
- 结束复盘按钮。

验收门禁：

- `idle`、`connecting`、`listening`、`assistant_speaking`、`error` 状态都可展示。
- 用户说话时不展示大面积纠错。
- 结束按钮可以进入复盘报告页。
- 不请求麦克风。
- 不连接 WebSocket。

### 阶段 9：课后总结 Mock 页面

目标：完善当前课后总结页面。

交付内容：

- 总分。
- 一句话结论。
- 能力拆解。
- 关键修正。
- 发音问题展示区。
- 语法 / 表达纠错展示区。
- 可复用表达。
- 复练同场景入口。

验收门禁：

- 报告 ready、empty、error 三种状态可展示。
- 页面重点是“下一次怎么说更好”，不是堆满数据。
- 可返回场景选择或进入历史记录。
- 不调用真实报告接口。

### 阶段 10：历史记录 Mock 页面

目标：完善当前历史记录页面。

交付内容：

- 历史记录列表。
- 历史详情占位入口。
- 从历史记录复练入口。

验收门禁：

- loading、empty、error 状态可展示。
- 历史项可进入报告详情占位。
- 历史项可作为复练入口。
- 不调用真实历史接口。

### 阶段 11：前端 client interface

目标：只定义后续后端对接所需的前端接口，不接真实网络。

交付内容：

- `ScenarioClient` interface。
- `SessionClient` interface。
- `ReportClient` interface。
- `HistoryClient` interface。
- `ProfileClient` interface。
- `RecommendationClient` interface。
- `RealtimeClient` interface。
- `AudioRecorder` interface。
- `AudioPlayer` interface。
- 对应 mock implementation。

验收门禁：

- 页面依赖 interface，不依赖具体 mock 文件。
- mock implementation 可以完整跑通前端流程。
- 真实后端对接时可以替换 implementation。
- 不写真实 baseURL、token、WebSocket 地址。

### 阶段 12：页面状态完善

目标：补齐所有前端状态。

交付内容：

- loading 状态。
- empty 状态。
- error 状态。
- retry 行为。
- disabled 行为。
- selected 行为。

验收门禁：

- 4 个页面都有 loading / empty / error 的合理展示。
- 错误态不导致导航卡死。
- disabled 按钮有明确视觉状态。
- retry 只调用 mock implementation。

### 阶段 13：移动端适配与可访问性

目标：保证页面在手机尺寸下可用。

交付内容：

- 小屏适配。
- 安全区适配。
- 文本换行检查。
- 按钮触控尺寸检查。
- 关键按钮 accessibilityRole。

验收门禁：

- 主要页面在常见手机宽度不溢出。
- CTA 文案不挤压或截断。
- 可点击区域不小于合理触控尺寸。
- 视觉层级在浅色和深色区域都可读。

### 阶段 14：前端 mock 全链路验收

目标：验证不依赖后端的前端完整流程。

交付内容：

- 场景选择 -> 实时对话 Mock -> 课后总结 Mock -> 历史记录。
- 从总结页复练同场景。
- 从历史记录生成复练入口。
- 所有页面状态演示。

验收门禁：

- 完整 mock 流程可以连续跑通。
- 没有 HTML 原型依赖残留。
- 没有乱码文案。
- 没有真实网络请求。
- `pnpm lint` 和 `pnpm exec tsc --noEmit` 通过。

### 阶段 15：后端对接交接文档

目标：为后续后端对接准备清单，不在当前阶段接后端。

交付内容：

- REST client 对接清单。
- WebSocket 对接清单。
- 音频采集与播放对接清单。
- 页面状态与接口状态映射表。
- mock 字段到接口字段映射表。

验收门禁：

- 文档写入 `docs/` 目录。
- 清楚标明哪些能力当前是 mock。
- 清楚标明哪些能力后续由后端或原生音频模块提供。
- 后端对接不需要重新理解前端页面结构。

## 4. 推荐里程碑

### 里程碑 A：前端视觉与结构确认

覆盖阶段 0 到阶段 7。

结果：

- 页面结构确认。
- 设计系统确认。
- 首页和配置页可演示。

### 里程碑 B：前端 mock 主流程闭环

覆盖阶段 8 到阶段 10。

结果：

- 不依赖后端即可演示完整学习流程。
- 实时对话、复盘、资产库均可展示。

### 里程碑 C：前端工程可替换性

覆盖阶段 11 到阶段 12。

结果：

- 页面通过 interface 依赖数据。
- mock implementation 可以被真实后端 implementation 替换。

### 里程碑 D：前端交付验收

覆盖阶段 13 到阶段 15。

结果：

- 移动端适配完成。
- mock 全链路验收完成。
- 后端对接清单完成。

## 5. 当前建议

先执行阶段 0 到阶段 10，完成一个基于当前页面、不依赖后端的前端 mock 版本。  
等页面结构、视觉、交互和状态都稳定后，再从阶段 11 开始做 interface 抽象和后端对接准备。
